# 目次重複・不要項目混入問題のデバッグ記録

## 問題の概要
EPUBファイルの読み込み時に、章目次の間に文中の段落番号が目次に含まれるようになった。
「衣笠彰梧_ようこそ実力至上主義の教室へ_第010巻.epub」で発生しており、文中に数字だけの行があることが理由の可能性がある。

## 調査の進捗
1. **コード解析**: 
   - `assets/reader.js` 内で、論理目次が機械的（`p-cover`等）な場合に本文内HTMLから `a[href]` を抽出して拡張目次（`enhancedToc` / `betterToc`）を作成する `tryExtractBetterTocFromHtml` 関数が動作している。
   - `tryExtractBetterTocFromHtml` は `landmarks`、`guide`、またはファイル名に `toc`, `contents`, `nav` を含むファイルを特定し、そこから `a[href]` を抽出している。

2. **原因特定**:
   - 参考EPUB（`衣笠彰梧_ようこそ実力至上主義の教室へ_第010巻.epub`）の `nav.xhtml`（論理目次）を抽出・解析。
   - 該当の `nav.xhtml` に、`○ 嵐 の 前 の 静けさ` などの章見出しの他に、本文中のアンカー付きで `１` や `２` などの数字のみの目次項目が同じ階層にフラットに登録されていることを確認。
   - これがそのままEPUB.jsを通じて論理目次として読み込まれ、UI上に描画されていたことが原因。

---

## 行った工程と対策
- **対策方針**:
  - 内部の章区切り計算や位置解決に影響を与えないよう、目次の描画（表示）フェーズでのみ「数字のみで構成される不要なラベル」を除外するフィルタリング処理を追加。
  - 書籍自体の章タイトルが意図的に数字のみ（「1」「2」「3」など）である場合に目次が消えてしまうのを防ぐため、**「テキストを含む有効な見出しが混在している場合のみ、数字のみの項目を除外する（混在判定）」** という安全策を導入。
- **実装内容**:
  - `assets/js/ui/renderers.js` に `isNumericLabel(label)` を定義し、半角・全角・漢数字・ローマ数字・丸数字等を判定。
  - `renderToc` において、事前に目次項目全体を走査してテキストの見出しが混在しているかを判定（`hasNonNumericLabel` フラグの抽出）。
  - `renderTocEntries` にフラグを渡し、混在している場合に限って数字のみの項目をスキップして描画。
- **テスト・検証**:
  - `scratch/test_filter.js` を作成し、混在ケース、全て数字のケース、ネストされた混在ケースで正しくフィルタリングが制御されることをシミュレーションテストで確認（合格）。
  - `node --check` コマンドで `renderers.js` の構文にエラーがないことを確認。
