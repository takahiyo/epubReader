# 開発ログ (dev_log.md)

## [2026-02-28] 継続スクロール（Join Mode）の実装

### 作業概要

`test_2.epub` のようなファイル分割形式のEPUBにおいて、スクロールモードで画像やテキストが途切れないよう、複数の章を一つのスクロールコンテナに結合して描画する「Join Mode」を実装する。

- **ステータス**: 完了

### 状況報告

- **現在の状況**: 実装完了。動作検証フェーズへ。
- **成功の境界線**: `test_2.epub` で章内の挿絵と本文が連続し、かつ表紙や他章との間には適切な区切り（改ページ）が入ること。
- **失敗の事象**:
  - 初期実装（全章結合）では、表紙と本文、あるいは異なる章同士が繋がってしまい、ユーザーの意図に反する表示となった。
  - `epubPaginator.js` のコード修正時に `replace_file_content` のマッチングに失敗し、複数回リトライが発生した。
- **失敗の根本原因**:
  - 単純な全章結合では「章の区切り」という論理構造を無視していたため。
  - ファイル内のインデントや改行コードの微妙な不一致、またはターゲット範囲の指定ミスによりツールが置換に失敗した。
- **実装内容**:
    1. `reader.js` に `generateSpineGroupsFromToc()` を追加し、目次から章の境界（spineIndex範囲）を特定。
    2. パジネーターに `spineGroups` を渡し、同一グループ内の「末尾以外」の spine item に `isJoined: true` を設定。これにより、グループの境界で自動的に `isJoined` が `false` となり、改ページが発生するようにした。
    3. `reader.js` の `renderEpubPage` を修正。現在のページが属するグループ全体の spine items を走査し、それらを連結して描画するようにした。
- **次のアプローチ**: Browser Agent による `test_2.epub` の動作実証。
