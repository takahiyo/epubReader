# Web小説 API デバッグログ 2

## 事象
Web小説の検索は成功するが、目次を開いた際に「エピソードが見つからない (Found 0 episodes)」というエラーが発生する。

## 行った工程
1. **正規表現の検証**:
   - `web-novel-provider.js` の `getTableOfContents` で使用しているエピソード抽出用の正規表現 `new RegExp('\/${ncode}\/(\\d+)\/?(?:$|\\?|#)', 'i')` を検証。
   - テンプレートリテラルのエスケープ仕様により、ブラウザ上での評価時にバックスラッシュが消え、意図しない正規表現になっていた可能性や、エッジケースに対応できていなかった可能性を考慮。

2. **抽出ロジックの改善 (2026-05-12)**:
   - 動的な `RegExp` の生成をやめ、`href.toLowerCase().indexOf('/ncode/')` および `substring`, 文字列の先頭からの抽出 `match(/^(\d+)/)` を用いるシンプルで堅牢なロジックに変更した。
   - デバッグ用に、抽出されたリンクの総数 `allLinks.length` をコンソールに出力するように修正。

## 仮説
もし上記修正後も 0 episodes となる場合、Codetabs（CORSプロキシ）を介したアクセスが「小説家になろう」側のボット対策（Cloudflare等）にブロックされ、エラー画面（HTML内にエピソードリンクが存在しない画面）が返却されている可能性が非常に高いです。
（その場合、リンク総数が極端に少なくなることで判別可能）
