### ⚠️ 2026-02-26 15:21 追加: **根本原因の特定と修正**

| 試行 | 内容 | 結果 |
| :--- | :--- | :--- |
| 1回目 | `handleAuthLogin` の `uiInitialized` 引数修正 + `auth:login` リスナー移動 | 効果なし |
| 2回目 | `syncAllBooksFromCloud` の `didApplyIndex` 判定改善 + 未リンク書籍対応 | 効果なし（`let index` 削除バグ混入） |
| 3回目 | `let index = {};` の宣言漏れを修正 | 効果なし（ReferenceError は解消したがプッシュ不発） |
| 4回目 | プッシュ判定で `_storage.data.cloudIndex`→生の `index` に変更 | **プッシュ成功**（7冊がD1に送信されたがプルが0件のまま） |
| **5回目** | **`remote.index` → `remote` 自体がインデックスデータであることを検出するよう修正** | **テスト待ち** |

**根本原因（第5試行で追加特定）**: Workerのレスポンスは `{cloudBookId1: {...}, cloudBookId2: {...}, ...}` というフラット形式だが、コードは `remote.index` でアクセスしていたため、`undefined` → `{}` となり、クラウドのインデックスデータが常に空として解釈されていた。

**修正内容**: レスポンス形式を自動判定し、`remote.index` が存在すればそれを使い、なければ `remote` 自体をインデックスデータとして扱うように変更。

**最終調整（2026-02-27）**:
- 進捗の競合解決において、日付が新しい場合でも「位置が全く同じ」であれば、プロンプトを出さずに自動で最新化するように調整しました（ユーザーの利便性のため）。位置が異なる（＝どちらで開くか意味がある）場合にのみダイアログが表示されます。
- **しおり一覧の拡張**: これまで「ローカルにファイルがある書籍」のしおりしか表示されていませんでしたが、`cloudIndex` に存在する未ダウンロード書籍のしおりも一覧に表示されるように修正しました。しおりをクリックすると、そのまま書籍のインポート（ファイル選択/リンク）画面に移行できます。
