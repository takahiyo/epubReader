# Web小説機能 引き継ぎ資料 (HANDOVER)

## 現在の状況 (2026-05-13 更新)
Web小説（小説家になろう・カクヨム）の閲覧機能は、基本的な読み込み・ナビゲーション・同期を含め安定動作する段階に達しています。

### 実装済みの主要機能
1. **ナビゲーション**:
   - 画面上下に配置された「前の話」「次の話」ボタンによる確実な章遷移。
   - `reader.js` の `next()` / `prev()` との統合。
2. **目次 (TOC) 連携**:
   - 検索時に取得した全エピソード情報をメインUIの目次パネルに表示。
   - 目次項目をクリックすることで該当エピソードへ直接ジャンプ可能。
3. **進捗同期とSSOT**:
   - クラウド同期に対応。Web小説特有の `location` オブジェクトを深層比較し、不要なコンフリクト警告を抑制。
   - `cloudState.js` (SSOT) により、進捗と位置情報が一元管理されている。
4. **自動タイトル生成**:
   - エピソードにタイトルがない場合、本文冒頭から自動抽出。

### 技術的な注意点
- **Location構造**: `reader.js` の `goTo` に渡す位置情報は `{ location: { location: episodeIndex, percentage: scrollPercentage } }` という形式を維持してください。
- **CORS対策**: 現在 `web-novel-provider.js` にて複数のプロキシを試行するロジックになっています。独自の Worker (BookReader Worker) が最優先される設定です。

## 今後の課題 (Next Steps)
1. **全文検索機能の強化**:
   - 現在の検索は「作品検索」のみ。開いている作品内での「本文検索」を、章範囲指定などで実現する必要があります。
2. **目次UIの最適化**:
   - 話数が多い作品（1000話超など）において、目次を全件表示するとレンダリングが重くなる可能性があるため、仮想リスト化やページネーションの検討が必要です。
3. **しおり機能の更なる統合**:
   - 複数端末間での「しおり（特定の文章へのマーク）」の同期精度向上。
